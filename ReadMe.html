thank you very much for choosing Inout Adserver ULTIMATE 6.0

This is version 6.0 of Inout Adserver Ultimate. To upgrade from any older version/edition of adserver/ppc engine, please follow the guidelines given here. After upgradation please go through the extended configurations and make necessary changes if required.

To install a fresh copy of the script, follow the steps 1-11 (  Click here to view instructions for upgradation. )
   
1. Create a new MySQL database using your website management control panel (eg: Cpanel) and add a user to the database.
   
2. Open the the admin/config.inc.php and edit some important configurations there as described below. Please ensure that editing the config.inc.php file is done using editors like notepad++(this is a free editor) which allow to save the file in utf-8 encoding without BOM.
   
3. Enter your license key in the config file.

NOTE
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
You can generate the license key after logging into the member area of inoutscripts.com.
Please use your email address and the given password to login.
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
   
4.

Now you can enter your MySQL settings in the config.inc.php file as described in it.

The value "localhost" for the setting $mysql_server need to be changed only if your host has a different name for the MySQL server.

The values for $mysql_username, $mysql_password and $mysql_dbname usually need to be entered like this in the config.inc file.

yourdomain_username
password just as it is
yourdomain_databasename

You can leave the value for $mysql_server_type as 1 itself.
   
5. Now, modify your admin area username and password in the config file. You can also change the admin login user and password anytime in future here in admin/config.inc.php.
   
6. Next you need to modify the value of $server_dir to point to the full path of the directory where you have installed the script,
eg. $server_dir="http://mydomain.com/"[Remember to include the last forward slash.]
   
7. Now you have to specify outgoing email settings . The value of setting $mailserver_type need to be modified only if mails are not being sent out. You can change this to 1 in such case. The value for $email_encoding need to be same as that of your adserver data encoding. The adserver data encoding can be configured from admin panel after installation.
By default adserver uses PHP mail() to send outgoing mails. Instead you may configure SMTP mailing settings if required from here. Please note that you must having PHP version > 5 for using SMTP mailing.
   
8. Change the access permission[CHMOD] of the directories ppc-banners,ad_logos ,credit-image, _cache, locale, admin/ppc-service-ads and admin/affiliate-banners to 777. In case of locale folder please ensure that permission is set recursively.
   
9. Set admin/adserver_cron.php as a cron job at a run frequecy of once an hour; If you have any doubts on setting a cron job, please contact our support desk, they will happily help you out.
   
10. Go to the URL http://yourdomain.com/admin/index.php in your browser and login using the username and password you set in the config.inc.php file.
Upon validating license key, mysql details, username and password the script will be automatically installed and will ask you to configure some important settings. You may configure it later if you wish, but it will not take more than a minute and we recommend you to do that along with the installation.

IMPORTANT NOTE
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
If you are getting any error message in the install page saying some loader file is missing,
please find the required file in the zip file here and upload to the 'ioncube' directory.
If you have any query, please contact our support desk. We will be glad to help you.
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
11. Backup Data Base:To use this you must enable mysqli and zip in your server. You can contact your server people to enable the same.
   
   
That's it! If you see some error messages in admin area, the installation may not be proper. You may contact us for help. Now you may read about some additional configurations and instructions below.
   
Extended Configurations
   
  Adserver 5.2 onwards, we are supporting some extended configurations which are avaliable in extended-config.inc.php in the installation folder. It contains configurations for renaming certain folders and caching of ad display. Also there is a configuration to put your adserver to maintenace mode with restricted ip access .
 
Look and Feel Management
   
1. You may manage the entire look and feel of the software by changing a few templates. All these templates can be found in the ppc-templates folder (advertiser specific templates) , publisher-templates folder (publisher specific templates)and common-templates folder (templates common to advertiser and single-sign-on specific templates). You can modify these html templates to look exactly the way you want to see it. You may not find only minimum PHP code in the templates pages. So it will be very easy for you to manage these templates. There are some template variables enclosed in {}. Keep those variables untouched. You may add whatever content you want to add to these templates.
   
2. I want to just modify the headers
You may want to change only the header of the page sometimes. The header of all pages without login authentication can be modified by changing common-templates/common-header.tpl.html. The header of login authenticated pages of advertiser is ppc-templates/advertiser-loggedin-header.tpl.html , of publisher is publisher-templates/publisher-loggedin-header.tpl.html and of single-sign-on accounts is common-templates/common-loggedin-header.tpl.html.
   
3. Changing Engine Name from 'Inout Adserver' to your Engine Name
Please login to the admin area using the username and password which you have set in the config.inc.php file. Then after logging in, under the basic configuration section you can find the option to change the engine name to your desired engine name. Also you need to modify the message corresponding to this engine name in different languages from under 'Languages'>'Edit Messages' menu.
   
4. I want to upload my logo
You can upload your logo from admin area Site Content > Public Page Settings . Also you have options to specify whether logo image alone or logo text alone or both should be displayed.
   
5. CSS control of all public pages is located in ppc-style.css in the root folder where you can make changes to link and font styles.
   
6. If you want to change the content of the mails to your advertisers,publishers you can modify the same in admin area under the section "Email Templates".
   
Upgradation General Instructions (  Click here to view instructions for upgrading from standard edition/ppc engine versions 2.2 or older. )
   
NOTE : If you had modified the templates in your previous installation , please note that you will have to do the modifications again with the new version templates.  
   
1. Do take a backup of your database by exporting it as a zip from phpmyadmin. If you have root access, you may take a dump using mysqldump tool.
   
2. Upload the admin/pre-check.php alone from the new pack and invoke the same in browser. If it gives some error, please fix the error before proceeding with further steps of upgration.
   
3. Upload all files except admin/config.inc.php and extended-config.inc.php . If there are some new settings present in the new config files please copy them to your old config files.
   
4. Invoke admin/install.php from browser. It will create new tables and alter existing ones without loosing any data.
   
Upgrading from standard edition/ppc engine versions 2.2 or older.
   
1. Follow step 1 and 2 of upgradation general instructions given above.
   
2. Copy the values of following settings from existing config.inc.php file to the new config.inc.php file $license_key, $mysql_server, $mysql_username, $mysql_password, $mysql_dbname, $username, $password, $server_dir
   
3. Upload files except admin/adserver_cron.php and run admin/install.php. It will create new tables and alter existing ones without loosing any data.
   
 

NOTE : If you are upgrading from versions prior to 2.0, basic ad blocks will be created during the installation time. The basic blocks are infact built from the banner sizes in your db. If you had created any custom banner sizes other than those which were present in the old installation, most likely they will be converted to banner only blocks. Also the maximum ad title length and maximum ad display url length are updated to 35 and maximum ad description length to 70. These settings have been made constants for the ease of ad display management. If any existing active ads exceeds this limit, their status will be updated to pending state.

   
4. Invoke admin/adserver-data-back-up-cron.php from browser (This may take some time to complete depending on your db size)
   
5. Invoke admin/show-statistics.php from browser. Go through the statistics which is sorted by time and verify that the raw data statistics is same as advertiser statistics/publisher statistics/referral statistics for every row. If they do not match, please try step 4-5 again. If they still don't match, contact us for further help.
   
6. If everything is fine you may now upload the admin/adserver_cron.php from the pack.
   
  NOTE : If you are upgrading from a versions prior to 2.0, then set admin/adserver_cron.php as a cron job at a run frequency of once every hour; If you have any doubts on setting a cron job, please contact our support desk, they will gladly assist you.
   
7. Ensure that the access permission[CHMOD] of the directories ppc-banners, credit-image, _cache, locale, admin/ppc-service-ads and admin/affiliate-banners is 777. In case of locale folder please ensure that permission is set recursively.
   
8. Finally go to the TO DO page from your admin area and do the pending tasks.
